# 05-MVP 实施清单

## Phase 1：项目骨架

- [ ] 初始化 Next.js 项目
- [ ] 接入 UI 基础组件和全局样式
- [ ] 配置 SQLite
- [ ] 配置本地图片存储目录
- [ ] 配置 Docker / NAS 部署方式

验收：本地可以启动页面，数据库可读写，图片目录可访问。

## Phase 2：生成工作台

- [ ] 两个 Tab：文生图 / 图生图
- [ ] prompt 输入
- [ ] 尺寸选择
- [ ] 数量选择：1 / 2 / 4，默认 1
- [ ] 模板选择
- [ ] 参考图上传
- [ ] 高级参数折叠区：负面提示词、参考强度、风格强度

验收：可以提交生成任务。

## Phase 3：sub2api image-2 调用

- [ ] 后端封装 sub2api 调用
- [ ] 前端不暴露 API Key
- [ ] 支持文生图
- [ ] 支持图生图
- [ ] 生成图片保存到 NAS 本地目录

验收：文生图、图生图都能生成图片并落盘；会话内继续修改统一走图生图。

## Phase 4：任务队列

- [ ] 任务表
- [ ] 后台 worker 轮询 queued 任务
- [ ] 状态更新：排队中 / 生成中 / 完成 / 失败
- [ ] 前端展示任务队列
- [ ] 支持多个任务连续提交

验收：连续提交多个任务，页面能看到状态变化。

## Phase 5：历史记录

- [ ] 时间倒序图片流
- [ ] 模式筛选
- [ ] 模板筛选
- [ ] 关键词搜索
- [ ] 下载图片
- [ ] 复制 prompt
- [ ] 再生成
- [ ] 用这张图生成

验收：生成后的图片能被复用。

## Phase 6：模板管理

- [ ] 内置用途模板
- [ ] 内置平台模板
- [ ] 公司自定义模板新增 / 编辑
- [ ] 从历史记录保存为模板

验收：模板能被工作台选择，并带出默认参数。

## Phase 7：管理统计

- [ ] 今日 / 本周生成次数
- [ ] 成功 / 失败次数
- [ ] 预估成本
- [ ] 热门模板排行

验收：管理视图能看到核心统计。

## MVP 完成标准

- [ ] 部署在 NAS 内网可访问
- [ ] 不需要登录即可使用
- [ ] 支持文生图、图生图
- [ ] 支持任务队列
- [ ] 支持历史复用
- [ ] 支持模板管理
- [ ] 支持管理统计
- [ ] 页面风格简约大气
